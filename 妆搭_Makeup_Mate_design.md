# 妆搭 Makeup Mate Design Spec

> 文件用途：专门约束 **妆搭 Makeup Mate** 的 UI 视觉风格、组件规范、交互规范，以及高保真 HTML 单页 Demo 的输出要求。  
> 适用对象：Claude Code / Codex / Cursor / 前端开发者 / UI 设计师。  
> 对应 PRD：`妆搭_Makeup_Mate_PRD_三页架构版.md`  
> 版本：V1.0

---

## 一、设计目标

### 1.1 产品气质

**妆搭 Makeup Mate** 不是传统美妆修图 App，也不是强电商导购工具，而是一个“会记住你的 AI 美妆视频陪练”。

因此，视觉风格应传达以下感受：

- 温柔；
- 干净；
- 可信；
- 有陪伴感；
- 有轻量 AI 科技感；
- 有美妆质感，但不甜腻；
- 有产品高级感，但不冷冰冰。

产品整体不要做成“黑色科技风”，也不要做成“高饱和粉色少女风”。  
最终方向应接近：

> **奶油白底 + 暖调奶茶粉 + 豆沙粉 + 少量勃艮第酒红强调。**

参考气质可以理解为：

- Aesop 产品页的克制感；
- 观夏式的东方温暖与留白；
- Apple 产品展示页的干净秩序；
- 美妆产品包装的柔和细腻；
- AI 应用的轻量智能感。

## 二、整体视觉风格

### 2.1 风格关键词

```text
奶油粉调
米白背景
燕麦白留白
暖调奶茶粉
脏粉强调
勃艮第酒红按钮
白色大圆角卡片
柔和阴影
胶囊输入框
苹果系统感
轻量 AI 感
女性向但不过度少女
美妆感但不电商促销
温暖但不甜腻
简洁但不空洞
```

---

### 2.2 不采用的风格

明确禁止以下方向：

```text
大面积黑色背景
纯荧光绿科技风
高饱和芭比粉
廉价粉色渐变
强电商促销风
二次元少女风
医美诊断报告风
后台管理系统风
PPT模板风
```

黑色可以用于：

- iPhone 手机边框；
- 灵动岛；
- 极少量文字强调；
- 状态栏细节。

但小程序界面主背景不使用黑色。

## 三、色彩系统

### 3.1 主色方案：奶油粉调

推荐采用以下色彩体系：

| 用途 | 色值 | 说明 |
|---|---:|---|
| 主背景 | `#FBF7F2` | 米白 / 燕麦白，作为全局背景 |
| 次背景 | `#F7EFE8` | 更暖的浅米色，用于页面分区 |
| 卡片背景 | `#FFFFFF` | 白色卡片，保持干净与高级 |
| 主色 | `#E8B4A0` | 暖调奶茶粉，用于主按钮、重点标签 |
| 次主色 | `#D89B8A` | 脏粉，用于 hover、次级强调 |
| 强调色 | `#7A2E2E` | 勃艮第酒红，用于关键 CTA、标题强调 |
| 浅粉底 | `#F8E2DA` | 标签背景、浅色气泡 |
| 浅杏色 | `#F3D6C8` | 柔和装饰色、卡片渐变 |
| 主文字 | `#2B211E` | 暖黑，不使用纯黑 |
| 正文文字 | `#5C4A43` | 温和深棕灰 |
| 辅助文字 | `#9A8378` | 弱提示、说明文字 |
| 边框 | `#EADCD4` | 卡片描边、输入框边框 |
| 分割线 | `#EFE4DE` | 页面细分隔 |
| AI 点缀 | `#B7E86B` | 极少量使用，表达 AI 激活状态 |

---

### 3.2 色彩使用比例

建议页面整体配色比例为：

```text
70%：米白 / 燕麦白背景
18%：白色卡片
7%：奶茶粉 / 脏粉
3%：勃艮第酒红
2%：AI 绿色点缀或状态色
```

注意：

- 勃艮第酒红只做强调，不要大面积铺满；
- AI 绿色只做小圆点、识别状态、进度激活，不要变成主色；
- 粉色要偏灰、偏奶茶，不要过亮；
- 背景要温暖，但要保证文字可读性。

## 六、页面视觉规范

## 6.1 首页

### 页面目标

首页负责：

> 粘贴链接 → 解析视频 → 生成妆容复刻卡片。

### 视觉要求

- 主背景使用 `#FBF7F2`；
- 顶部清爽，保留足够留白；
- 主标题要有温柔但坚定的产品感；
- 链接输入框使用白色大圆角胶囊；
- 主按钮使用勃艮第酒红或奶茶粉；
- 解析结果卡片使用白底、大圆角、柔和阴影；
- 标签使用浅粉底 + 深棕文字；
- 不要做成表单页或后台页。

### 首页核心文案

```text
把刷到的美妆视频
变成适合你的专属上妆教程
```

### 首页组件

```text
顶部品牌区：
妆搭 Makeup Mate

链接输入区：
粘贴抖音美妆视频链接

主按钮：
开始解析妆容

解析状态：
正在提取关键帧
正在识别妆容风格
正在拆解上妆步骤
正在生成复刻卡片

妆容卡片：
清冷感通勤妆
低饱和 / 干净 / 淡颜友好
难度：中等
耗时：18 分钟
适合：通勤 / 上课 / 面试

操作按钮：
分享卡片
导入聊天
```

---

## 6.2 聊天页

### 页面目标

聊天页是产品核心体验页，负责：

> 接收妆容卡片 → 文字/语音/视频陪伴上妆 → 上传照片分析个人风格 → 查看历史记录。

### 视觉要求

- 聊天页仍然使用浅色体系，不使用黑色背景；
- 可在顶部区域加入轻微玻璃拟态；
- 右上角必须保留两个功能入口：
  - 左侧：历史记录；
  - 右侧：上传照片 / 分析个人风格。
- 底部输入框采用大胶囊设计；
- AI 消息使用浅粉底；
- 用户消息使用白色卡片；
- 当前步骤卡片使用白底 + 奶茶粉顶部提示；
- 语音/视频陪伴状态使用小面积 AI 绿色点缀。

### 聊天页顶部布局

```text
左侧：返回
中间：妆搭 / MM
右上角左侧：历史记录图标
右上角右侧：上传照片图标或用户头像
```

### 聊天页底部输入框

```text
左侧：图片上传图标
中间：发送消息或上传图片
右侧：语音按钮 / 发送按钮
```

输入框建议：

- 背景：`#FFFFFF`
- 边框：`#EADCD4`
- 圆角：999px
- 阴影：轻微暖色阴影
- 发送按钮：勃艮第酒红或奶茶粉圆形按钮

## 6.3 我的页

### 页面目标

我的页负责展示：

> 用户的专属妆容档案、AI 已记住的偏好、历史复刻总结、隐私设置。

### 视觉要求

- 我的页要像“个人风格说明书”，不要像医院体检报告；
- 顶部可以使用柔和头像区；
- 档案卡片要精致、清晰、可扫读；
- 标签要有温暖感；
- 重要标签用勃艮第酒红强调；
- 不要出现“颜值评分”；
- 不要用“缺陷判断”表达。

### 我的页核心模块

```text
顶部：
头像 / 昵称 / 妆容档案完成度

我的妆容档案：
方圆脸
自然偏暖
淡颜偏自然
内双
适合上移腮红
适合后半段短眼线
适合奶茶豆沙唇色

AI 已记住：
不喜欢浓眼妆
上班日妆容控制在 15 分钟内
眼线需要新手版
适合低饱和唇色

隐私设置：
仅本次分析
保存为妆容档案
删除照片分析
清空历史记忆
```

## 八、动效与交互规范

### 8.1 动效原则

动效应当：

- 轻；
- 慢；
- 柔；
- 不抢内容；
- 有 Apple-like 的精致感。

禁止：

- 夸张弹跳；
- 大幅旋转；
- 强烈闪烁；
- 复杂粒子效果；
- 网页游戏式动效。

---

### 8.2 推荐动效

| 场景 | 动效 |
|---|---|
| 页面切换 | 轻微淡入 + 8px 上移 |
| 卡片出现 | fade-in + translateY |
| 按钮点击 | scale(0.98) |
| 解析中 | 小圆点脉冲 |
| 导入聊天 | 卡片轻微滑入聊天区 |
| 标签出现 | 轻微 stagger 延迟 |
| 手机模型 | 桌面端轻微悬浮阴影 |

---

### 8.3 交互反馈

所有可点击元素必须有反馈：

```text
hover：亮度轻微提升或阴影增强
active：scale(0.98)
disabled：透明度 0.5
loading：按钮显示解析中
```

## 十三、最终设计结论

**妆搭 Makeup Mate** 的视觉方向应确定为：

> **奶油白 / 燕麦白背景 + 暖调奶茶粉主色 + 脏粉辅助 + 勃艮第酒红强调 + 白色大圆角卡片 + 胶囊输入框 + 轻量 AI 绿色点缀。**

这套风格比黑色背景更适合产品的“陪伴”属性，也比高饱和粉色更克制高级；既有美妆感，又不会过度女性化，更适合黑客松路演、产品展示和后续真实小程序开发。
