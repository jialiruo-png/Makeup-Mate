## 三、项目简介

**妆搭 Makeup Mate** 是一款 AI 美妆复刻陪练小程序。

它解决一个很高频但被忽视的场景：**用户在抖音、小红书刷到好看的妆容，收藏、截图、暂停反复看，但真正动手画的时候——看不懂步骤、找不到重点、不知道适不适合自己、化到一半没人指导。**

妆搭把这个断点闭合：

- 用户在首页**粘贴美妆视频链接，或上传妆容图片、本地短视频**，AI 自动拆出妆容的**步骤、产品类型、预计耗时、翻车点**，生成一张可分享、可导入聊天的「**妆容解析卡片**」。
- 进入聊天页后，AI 化身一个**陪练搭子**，用文字 / 语音 / 拍照 / 视频陪用户一步步完成上妆；如果用户上传过自拍，AI 会基于脸型、肤色、五官把原教程**改写成「我的版本」**。
- 没有明确想画的视频时，用户可以从**美妆灵感库**——博主灵感、风格复刻、场景妆容、新手陪练四个入口——开始探索。
- 每一次复刻都会沉淀到「**我的妆容档案**」：MM 记得用户喜欢什么、不擅长什么、上次画到哪一步，**用得越多，越懂你**。

> 一句话：不是又一个修图工具，也不是虚拟试妆，**是把短视频拆给用户看，再陪用户画完**。

## 四、项目背景

### 4.1 市场观察

抖音、小红书、B 站等平台已经沉淀了海量美妆教程内容，用户消费意愿极高。但**消费内容 ≠ 复刻得了内容**，存在显著的执行断点：

| 用户行为 | 现实结果 |
| -------- | -------- |
| 看完视频很心动 | 收藏夹里多了一条 |
| 想复刻 | 反复暂停、记不住产品名、看不出哪一步是关键 |
| 真动手画 | 找不到对应产品 / 不知道顺序 / 卡在某一步 |
| 画完发现 | 不像、显凶、显脏、显假，最后涂掉 |

**美妆视频成了内容消费，不是可执行方案。** 尤其对新手、学生党、上班族这类「想画但不专业」的人群，这个断点的代价是直接放弃。

### 4.2 目标用户

| 用户类型 | 典型痛点 | 妆搭提供的价值 |
| -------- | -------- | -------------- |
| 美妆新手 | 看得懂结果，看不懂过程 | 拆解步骤，陪伴执行 |
| 学生用户 | 想尝试流行妆容，工具与预算有限 | 推荐替代产品和低门槛画法 |
| 上班族 | 早八 / 通勤 / 面试，需要快速妆 | 生成时间可控的场景方案 |
| 美妆视频重度用户 | 收藏多，复刻少 | 把收藏视频变成可执行卡片 |
| 手残党 | 眼线、修容、腮红总翻车 | 降难度改写 + 实时纠错 |

### 4.3 竞品分析

当前市场上与美妆相关的产品大致分四类，每类都有局限：

| 类型 | 代表产品 | 核心能力 | 局限 |
| ---- | -------- | -------- | ---- |
| 修图美颜类 | 美图秀秀、醒图 | 图片精修、人像美颜、妆容模板 | 偏「修图结果」，不解决真实上妆过程 |
| AI 测肤 / 测脸类 | 美图宜肤、美的你 | 肤质分析、脸型测试、色彩测试 | 偏测评推荐，缺少视频教程复刻闭环 |
| AR 试妆类 | YouCam Makeup、ModiFace | 虚拟试色、AR 妆效预览 | 偏电商导购，较少关注用户真实操作 |
| 内容平台类 | 抖音、小红书、B 站 | 大量美妆教程内容 | 内容非结构化，复刻成本高 |

它们各自解决「修得好看」「试个色号」「测脸型肤色」「看更多内容」，但**没人完整解决「刷到一个妆 → 我能画出来」这个端到端流程**。

### 4.4 妆搭的差异化定位

| 维度 | 传统产品 | 妆搭 Makeup Mate |
| ---- | -------- | ---------------- |
| 核心对象 | 图片、人脸、商品色号 | 美妆视频、图片、达人风格、场景妆容 |
| 用户动作 | 修图、试色、测肤 | 粘贴链接 / 上传图片视频 / 跟着 AI 上妆 |
| AI 角色 | 修图工具、识别工具、导购工具 | 内容拆解员、个人化妆师、陪练搭子 |
| 输出 | 精修图、试妆图、商品推荐 | 妆容解析卡片、个性化步骤、陪伴上妆记录 |
| 长期价值 | 模板复用 | 记住妆容档案与偏好，越用越懂你 |

妆搭不与美图、醒图竞争「谁修图更自然」，不与 AR 试妆工具竞争「谁试色更逼真」。妆搭建立的是**新的产品心智**：

> **AI 美妆视频复刻陪练。**

### 4.5 我们的核心优势

| 维度 | 优势 |
| ---- | ---- |
| **内容可执行化** | 抖音视频关键帧 + 字幕 + 评论一起读，拆出步骤、产品、时间、翻车点。让短视频从「看着像简单」变「真的能画」 |
| **建议个性化** | 上传一张自拍，AI 分析脸型、肤色、五官，把原教程改写成「我的版本」（例：「上挑眼线 5mm → 后半段 2mm」） |
| **陪伴式上妆** | 文字 / 语音 / 拍照 / 视频通话四种方式，遇到问题随时换种说法、找替代产品、拍照检查 |
| **妆容档案长期化** | MM 记住你不喜欢浓眼妆、上班只想 15 分钟、画腮红总偏低，下一次直接给你你的方案 |
| **奶油粉调高级视觉** | 不走少女粉，不走深色科技，走 Aesop / 观夏 / Apple Notes 的克制感，目标受众接受度高 |
| **隐私优先** | 默认不保存原始自拍和视频，仅保存结构化标签；不做人脸识别、不建立人脸库 |

### 5.3 首页 · 解析入口

首页只做一件事：**把一个美妆内容来源，变成一张可执行的妆容解析卡片**。

三种入口：

1. **上传图片**（默认）—— 妆容截图、博主妆容图、想复刻的妆容照片
2. **上传视频** —— 本地短视频片段，系统抽帧分析
3. **粘贴链接** —— 抖音美妆视频链接，AI 自动抽帧解析

点击「开始解析妆容」后，AI 走 4 步流程：

```
✓ 读取内容来源
✓ 提取画面关键帧
○ 识别妆容风格
○ 生成解析卡片
```

![首页 · 三入口 + 解析过程](picture/解析博主妆容.png)

#### 生成的「妆容解析卡片」包含

- **妆容名称**（例：雀斑奶油妆）
- **风格标签**（低饱和 / 自然 / 伪素颜 / 淡颜友好）
- **复刻难度** / **预计耗时**
- **产品类型**清单（气垫、浅棕眼影、奶茶豆沙唇泥、杏粉腮红、雀斑笔……）
- **步骤拆解**（底妆 / 眼妆 / 腮红 / 唇妆 / 雀斑点缀，每步都有具体描述）
- **翻车点提示**（雀斑点得太均匀会显假 / 腮红过重容易显脏 / ……）

卡片可以**分享**给朋友，或**导入聊天**让 AI 陪你画。

### 5.4 聊天页 · AI 陪伴 + 美妆灵感库

聊天页是产品的核心体验页。它承担两件事：

1. **AI 陪练**：基于妆容解析卡片或灵感库选择，陪用户一步步画完一个妆
2. **美妆灵感库**：没有明确想画的视频时，从达人 / 风格 / 场景 / 新手陪练四个入口探索

#### 5.4.1 AI 陪练

- 用户消息支持**文字、语音、拍照、视频通话**
- AI 基于妆容卡片给出**当前 Step + 操作要点**
- 快捷按钮：「我完成了」「换种说法」「没有这个产品怎么办」「拍照检查」
- 卡住时，AI 会用更简单的语言重新解释，或推荐平价替代产品
- 用户拍照检查时，AI 评估眼线长度、腮红位置、底妆厚度

![AI 陪练对话](picture/ai对话语音视频陪练.png)

#### 5.4.2 美妆灵感库

四个分类入口（下划线 segmented tab，纯文字 + 短线指示）：

| 入口 | 包含内容 |
| ---- | -------- |
| **博主灵感** | 已收录 12 位美妆博主，按教程型 / 风格型 / 仿妆型 / 改造型 / 测评型分类 |
| **风格复刻** | 8 大妆容风格：入门、日韩、欧美、亚裔、日常、纯欲、港风、Cos |
| **场景妆容** | 通勤 / 上课 / 约会 / 面试 / 拍照 / 聚会 / 证件照 |
| **新手陪练** | 第一次画眼线 / 第一次画腮红 / 10 分钟新手妆 等低难度训练 |

支持**搜索博主 / 风格**，支持按**风格 / 难度 / 场景**多维筛选。

每张灵感卡片有两个操作：
- **生成我的版本** —— 调取个人档案改写成专属方案
- **导入聊天** —— 直接开启 AI 陪练

### 5.5 我的页 · 个人妆容档案（工作台风格）

我的页是一份**用户长期美妆记忆**，参考 App 后台工作台的工整感。

#### 5.5.1 头部信息区

- 标题「我的妆容档案」 + 「重新分析」按钮
- 头像 + 用户名 + 等级徽章（新手 · L2）
- 「加入第 24 天 · 已复刻 12 次妆容」
- **妆容档案完成度**：3px 细进度条 + 百分比

#### 5.5.2 3 个 stat 速读卡

| 脸型 | 五官 | 色调 |
| ---- | ---- | ---- |
| 方圆脸 | 淡颜偏自然 | 自然偏暖 |

#### 5.5.3 面容密码（时间线分析）

左侧 ○ + 细竖线串联两个分析章节。

**面容密码** 包含 5 个子项：

- **脸型判断**：方圆脸，下颌饱满圆润，颧骨柔和，下颌线较短，整体轮廓偏温柔
- **五官风格**：淡颜偏自然，五官分布舒展，眉眼柔和不锐利
- **眼型分析**：内双眼，建议眼妆走「后半段短眼线 + 浅棕铺色」路线
- **修饰重点**：腮红上移到眼下外侧，视觉拉长面中
- **肤色特征**：自然偏暖肤色，搭配奶茶豆沙等柔雾色系最贴肤

#### 5.5.4 色彩标签（实色色块）

- **适合唇色**：奶茶色 / 豆沙色 / 低饱和红棕
- **适合腮红**：杏粉 / 豆沙粉
- **适合眼影**：浅棕 / 大地色

每个色块用实际颜色填充，所见即所得。

#### 5.5.5 其余信息

- **建议避开**：重修容 / 欧美挑眉 / 过长上挑眼线（虚线描边降级标签）
- **MM 记住的事**：4 条用户偏好（不喜欢浓眼妆 / 上班 15 分钟内 / 眼线后半段 / 没有修容盘……）
- **最近的复刻**：小圆点时间线，3 条最近记录
- **统计**：已复刻 12 次妆容，覆盖 3 类风格，最近一次在 5 月 22 日
- **CTA**：试试一键生成「我的版本」（酒红主按钮）
- **隐私 footer**：仅本次分析 / 删除分析记录 / 清空妆容记忆

## 六、技术架构

### 6.1 技术栈

| 层 | 技术选型 |
| -- | -------- |
| **前端** | Vite · React 18 · TypeScript · CSS Modules |
| **后端** | FastAPI · SQLAlchemy · Pydantic |
| **数据库** | PostgreSQL · 阿里云 OSS（媒体资源） |
| **AI 模型** | Qwen-VL-Max-Latest（视觉解析 / 抽帧识别）· Claude（对话陪练 / 风格改写） |
| **部署** | Docker · uvicorn |

### 6.2 关键技术点

- **视频/图片解析**：调用 Qwen-VL 视觉模型，输入图片或抽帧后的关键帧，输出结构化的妆容标签 + 步骤 + 产品类型 + 翻车点
- **聊天历史**：所有会话落 PostgreSQL，每次发送时把最近 10 轮上下文 + 当前妆容卡片一起喂给 Qwen，避免 AI 失忆
- **个性化改写**：将用户妆容档案（脸型、肤色、偏好、避免风格）作为 system prompt 注入，让 AI 输出「我的版本」而非通用方案
- **响应式适配**：桌面端展示手机外框 + 品牌区，移动端（≤768px）自动脱掉外框全屏铺满，适配 iPhone 刘海 + Android safe-area
- **PWA 支持**：viewport-fit=cover + apple-mobile-web-app-capable，添加到主屏可全屏体验

### 6.3 核心接口

| 接口 | 用途 |
| ---- | ---- |
| `POST /makeup-cards/analyze` | 接收链接 / 图片 / 视频，返回妆容解析卡片 |
| `POST /media/upload` | 上传图片或视频，返回 mediaAssetId |
| `POST /chat/sessions` | 创建 AI 陪练会话 |
| `POST /chat/sessions/{id}/messages` | 发送消息 + 接收 AI 回复（支持图片附件） |
| `GET /chat/sessions/{id}/messages` | 拉取会话历史 |
| `GET /profile/me` | 获取用户妆容档案 |
| `POST /profile/analyze-photo` | 上传自拍，分析个人风格 |
| `GET /history` | 获取历史复刻记录 |

## 七、产品边界与合规

### 7.1 不做的功能（V1 不做）

- ❌ 颜值评分
- ❌ 医疗诊断 / 皮肤病判断
- ❌ 容貌缺陷评价
- ❌ 强商品导购
- ❌ 大而全社区
- ❌ 美甲、穿搭、发型等泛化功能
- ❌ 精准人脸识别或身份识别

### 7.2 表达边界

| ❌ 错误表达 | ✅ 正确表达 |
| ----------- | ----------- |
| 你的脸太圆，眼睛不够大，颜值偏低 | 你的脸型偏圆，更适合把腮红位置上移到眼下外侧，视觉上更轻盈；眼妆建议短眼线和浅色晕染，保留干净感 |

### 7.3 隐私原则

1. **默认不保存原始自拍和视频**
